# Интерфейс для агентов

Шаг 2 в [[Конспект проектирование под агентов|стеке]]: спроектировать **идеальный интерфейс для агентов — раньше, чем для людей**. В этом и есть переворот: машина в очереди первой.

> [!quote]
> Спроектируйте идеальный интерфейс для агентов.

## Суть
Раз прямой пользователь — [[Агент как пользователь|агент]], то именно ему нужен «идеальный» вход: чистые, описанные [[Универсальные примитивы|примитивы]], которые модель вызывает уверенно и без догадок. Хороший агентский интерфейс — это понятный контракт: что можно сделать, что вернётся, что пошло не так.

## Подробнее
- Агент «видит» не пиксели, а структуру: имена, параметры, ответы, ошибки. Их и проектируем.
- Описания операций — часть интерфейса: модель действует по тексту, а не по интуиции.
- Предсказуемость важнее «красоты»: одинаковое поведение в похожих случаях, явные ошибки вместо тихих сбоев.
- Практическое воплощение этого слоя в индустрии — MCP, системы разрешений, вызовы инструментов (см. [[ИИ как инфраструктура]]).

> [!tip] Что запомнить
> Идеальный интерфейс для агента — это честный контракт над хорошими примитивами: предсказуемый вызов, понятный ответ, явная ошибка.

> [!check] Практический вывод
> Проектируй API/инструменты так, чтобы их можно было корректно вызвать «вслепую», по одному описанию. Это и есть тест на агент-готовность.

## Связи
- [[Агент как пользователь]] — для кого этот интерфейс.
- [[Универсальные примитивы]] — то, что интерфейс обнажает.
- [[Интерфейс для людей]] — следующий шаг, поверх агентского.
- [[ИИ как инфраструктура]] — обвязка (MCP, инструменты, разрешения) живёт именно здесь.
