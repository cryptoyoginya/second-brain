# Три уровня generative UI

Шаг 4 конвейера — сборка интерфейса под интент. Три уровня зрелости по нарастанию гибкости и цены: выбрать готовый экран → собрать из проверенных компонентов → сгенерировать код на лету. Практический выбор — почти всегда средний.

> [!quote]
> LLM не рисует пиксели — она возвращает структуру: «график продаж + таблица топ-10 + фильтр по датам», а движок рендерит из проверенных компонентов.

## Суть
- **Уровень 1 — выбор готового экрана.** Под каждый интент заранее сделан экран, система открывает нужный с подставленными параметрами. Дёшево, предсказуемо, негибко. Так работает большинство голосовых ассистентов.
- **Уровень 2 — сборка из компонентов (generative UI).** Библиотека готовых блоков (график, таблица, фильтр, карточка, форма); LLM возвращает **структуру** (что показать), движок рендерит из проверенных кирпичей. Vercel AI SDK (`streamUI`), виджеты ChatGPT и Claude. Сейчас мейнстрим — баланс гибкости и надёжности, единый стиль.
- **Уровень 3 — генерация кода на лету.** Модель пишет настоящий HTML/JS под запрос. Максимум гибкости, но дорого, медленно, непредсказуемо (сегодня кнопка тут, завтра там). Пока — для прототипов (v0, Claude Artifacts), не для продакшн-интерфейсов.

## Подробнее
- Почему уровень 2 — золотая середина: гибкая компоновка сверху, но каждый кирпич надёжный, стилистически согласованный и протестированный. Модель отвечает за *что показать*, а не за *как отрисовать*.
- Уровень 3 ломается ровно там же, где [[Интерфейс как карта возможностей|адаптивные меню]]: непредсказуемость расположения → нет мышечной памяти, нет доверия.
- Все три опираются на [[Реестр возможностей (tool use)|реестр]]: без формального «что система умеет и какие данные есть» собирать нечего.
- Инженерные боли (общие для интент-UI): **fallback** при непонятом запросе (уточняющий вопрос или обычное меню); **латентность** (LLM = секунды vs мгновенный клик); **доверие** (почему показали именно это + стабильный «запасной выход» в классический UI).

> [!tip] Что запомнить
> Готовый экран (дёшево/жёстко) → сборка из компонентов (гибко/надёжно, мейнстрим) → генерация кода (гибко/дорого/непредсказуемо, пока прототипы). LLM возвращает структуру, движок рендерит проверенные кирпичи.

> [!check] Практический вывод
> По умолчанию целиться в уровень 2: библиотека надёжных компонентов + LLM, отдающая структуру. Уровень 3 держать для прототипов, не для продакшна. Заложить fallback, бюджет на латентность и «запасной выход» в классику. Пет-проект на выходные: чат-поле + 3–4 инструмента + ~5 React-компонентов + function calling — проходит всю цепочку.
