# Конспект: интент-интерфейсы

Разбор идеи «сказал цель — получил интерфейс»: с одной стороны — критика (три места, где идея ломается), с другой — как это реально устроено под капотом (конвейер из четырёх шагов). Продолжение куста [[Конспект проектирование под агентов|проектирования под агентов]]: там агент стал прямым пользователем, здесь — что происходит с интерфейсом человека, когда взаимодействие идёт по намерению, а не по командам.

> [!abstract] Вся суть в трёх строках
> 1. **Сдвиг парадигмы** (Нильсен): от command-based (говорю *как*) к [[Интент вместо команды|intent-based]] (говорю *что хочу*, интерфейс собирается под это).
> 2. **Три слабых места:** affordances так не отменить ([[Affordances и сигнификаторы|они генерируются]], а не исчезают); [[Цель формируется в процессе|человек часто не знает цели]]; статичный UI — это [[Интерфейс как карта возможностей|карта того, что система умеет]], и она теряется.
> 3. **Будущее гибридное:** стабильное предсказуемое ядро + intent-слой сверху (генеративный command palette), а не полностью текучий интерфейс.

## Конвейер: от «сказал цель» до «появился экран»
Между намерением и экраном — четыре шага (+ обратная петля уточнений):

1. **Ввод намерения.** Текст/голос, но интент можно и *выводить* без слов — из контекста (время, документ), поведения (три раза открыл отчёт → предложить), роли (бухгалтеру и маркетологу разные экраны). Живой минимум — command palette (Cmd+K в Notion, Figma, VS Code).
2. **Распознавание.** Раньше — NLU-классификаторы (Dialogflow, Rasa): руками список интентов + слоты-параметры, так работают Siri/Alexa. Сейчас — LLM: даёшь фразу + описание возможностей, она сама выбирает интент и параметры. Гибче, не надо перечислять все формулировки.
3. **[[Реестр возможностей (tool use)|Подбор возможностей]].** Формальный список: какие функции, данные, параметры. В мире LLM — tool use / function calling (JSON-описания инструментов). Главное: реестр кто-то должен спроектировать — магия не в генерации, а в машиночитаемо описанных возможностях.
4. **[[Три уровня generative UI|Сборка интерфейса]].** Три уровня зрелости: готовый экран под интент → сборка из проверенных компонентов (generative UI, Vercel AI SDK / виджеты ChatGPT и Claude) → генерация кода на лету (v0, Artifacts; пока для прототипов).

**Обратная петля — самое важное.** «Нет, за прошлый месяц», «добавь разбивку по регионам». Интент-интерфейс — это диалог, а не одноразовый запрос: цель [[Цель формируется в процессе|дозревает по ходу]].

## Три критических уточнения
- **Affordances vs signifiers.** Affordance — не элемент дизайна, а отношение «возможности системы ↔ способности человека». Кнопку можно не рисовать, но возможность «фильтровать по дате» либо есть, либо нет. Адаптивный UI не отменяет affordances — [[Affordances и сигнификаторы|генерирует их на лету]]; меняются сигнификаторы: из статичной раскладки → в контекстные.
- **Человек не знает цели.** «Хочу приколов» — не цель, а настроение; цель [[Цель формируется в процессе|формируется в процессе]] (Сачман, situated action, 80-е). Заставить артикулировать намерение — само по себе трение.
- **Обучаемость и привычка.** Статичный UI — [[Интерфейс как карта возможностей|карта возможностей]]; под запрос попросишь только то, что вообразил. Ломается мышечная/пространственная память (Microsoft, адаптивные меню Office 2000).

## Где идея сильна
Профессиональный софт с тысячами функций, где никто не помнит, где что лежит («анализ аналитики по проекту xyz»): цель формулируется явно, дашборд собирается под неё. Массовый потребительский сценарий с размытым намерением — хуже.

> [!tip] Что запомнить
> Intent-based UI — не «магия», а конвейер: ввод → распознавание → реестр возможностей → сборка. Слабые места — не в технологии, а в человеке (не знает цели, теряет карту возможностей). Ставка не на текучий интерфейс, а на гибрид: предсказуемое ядро + генеративный intent-слой.

> [!check] Практический вывод
> Пощупать всю цепочку за выходные: чат-поле + 3–4 описанных инструмента (function calling) + библиотека из ~5 React-компонентов + вызов LLM. Инженерные боли заложить сразу: fallback при непонятом запросе, латентность (LLM = секунды vs мгновенный клик), доверие (почему показали это + стабильный «запасной выход» в классический UI).

- [[Три уровня generative UI]] — готовый экран / сборка из компонентов / генерация кода.

## Дополнительно изучить
- Я. Нильсен, «AI: First New UI Paradigm in 60 Years» — первоисточник тезиса о смене парадигмы (в его же тексте — оговорка про выживание GUI, см. [[Интерфейс как карта возможностей]]).

- Л. Сачман, «Plans and Situated Actions» (1987) — situated action, база критики «человек знает цель».
- Vercel AI SDK (`streamUI`), Claude/ChatGPT widgets — рабочие примеры generative UI уровня 2; v0, Claude Artifacts — уровень 3.
