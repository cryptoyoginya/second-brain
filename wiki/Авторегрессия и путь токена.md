# Авторегрессия и путь токена

Как LLM набирает ответ: она генерирует по **одному токену за раз**, каждый раз перечитывая весь текст заново. Отсюда и «печатает по словам», и вся экономика — весь путь прогоняется на каждый токен.

> [!quote]
> Модель пишет, перечитывая себя. Один токен — один полный проход сквозь все слои. Потом снова, с учётом того, что только что написала.

## Суть
Путь одного токена сквозь модель:
**[[Эмбеддинги|эмбеддинг]] → десятки слоёв ([[Внимание (attention)|внимание]] + feed-forward) → вероятности → выбор**
- **Вероятности** — финальное представление сравнивается со всеми ~100k токенами словаря: у каждого своя вероятность быть следующим.
- **Выбор** — с долей случайности. **Temperature**: высокая = мягче/разнообразнее, низкая = жёстче/предсказуемее.
- **Авторегрессия** — выбранный токен дописывается ко входу, и весь путь прогоняется ЗАНОВО для следующего. Поэтому ответ набирается постепенно.

## Подробнее
- Отсюда [[Стриминг ответа|стриминг]]: токены отдаются по мере генерации, не дожидаясь конца.
- Отсюда же стоимость и латентность: длинный ответ = много полных проходов; входной контекст перечитывается каждый раз (это и делает [[Беспамятность и контекстное окно|длинный контекст]] дорогим).
- **Temperature — продуктовая ручка:** для структурных/детерминированных фаз ставят низкую (предсказуемость, форма), для творческого текста — выше (разнообразие). Это не «ум», а ширина разброса при выборе.

> [!tip] Что запомнить
> LLM генерирует по одному токену, каждый раз перечитывая всё заново (авторегрессия). Выбор — вероятностный, temperature задаёт разброс. «Печатает по словам» — прямое следствие устройства.

> [!check] Практический вывод
> Две ручки под контролем продакта: длина (каждый токен = деньги и время, короткий ответ дешевле и быстрее) и temperature (низкая для форм/данных, высокая для креатива). Нужна строгая форма ответа — не полагайся на temperature, задай [[Фазы пайплайна и гейты|схему]].
