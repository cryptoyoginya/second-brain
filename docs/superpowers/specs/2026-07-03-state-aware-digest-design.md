# Спека: State-aware двухконтурный дайджест + проактивный Scout

- Дата: 2026-07-03
- Статус: утверждён дизайн, ждёт ревью спеки → план реализации
- Контекст: `CLAUDE.md` (операции Digest/Scout), MCP-сервер `health-kb` (агент-коуч), вики второго мозга.

## 1. Проблема и цель

Обычный «дайджест под состояние» банален: состояние лишь подкрашивает список чтения («устала → мягче»). Это **single-loop** — оптимизация внутри существующей рамки. Корреляционный майнер («делаешь X → метрика Y растёт») — тот же single-loop в обёртке.

**Цель:** дайджест, который работает на уровне **double-loop learning** (Аргирис–Шён) — ставит под вопрос саму рамку (ментальные модели, скрытые правила, цели, ценности), опираясь на уникальную связку двух баз:
- **health-kb** = улики поведения и исходов (сон, recovery, strain, настроение, `05_decisions`);
- **wiki** = рамки Кристины (её страницы-модели и правила);
- **[[Обо мне]]** = цели, ценности, Фокус.

Плюс — проактивный **Scout**: система сама приносит разноформатные источники (статьи, эссе, книги, видео, лекции, сайты), привязанные к обнаруженной рамке, граунженные реальным веб-поиском.

## 2. Принципы

1. **Double-loop как хребет.** Single-loop допустим только тонкой строкой «тактика», честно помеченной.
2. **Только по уликам.** Двухконтурный блок срабатывает лишь при конкретном обнаруженном расхождении данных health-kb с рамкой/правилом/целью. Нет улик — дайджест тихий и короткий, без выдуманной глубины.
3. **Прямо, без смягчения.** Если улика есть — называть расхождение прямо и неудобно.
4. **Конкретика, не полосы.** Матч под реальную сегодняшнюю связку, а не общие «мало/много энергии».
5. **Тренд, не точка.** Улика живёт в траектории (~3–7 дней), а не в одном дне.
6. **Приватность взаимна.** Контент health-kb — только в ответе/`.context/`; никогда не пишется в `wiki/` и не коммитится. Ссылки на health-kb — по слою/файлу, конкретика локальна.
7. **Не наполнять.** Каждая строка либо кормит рамку/действие, либо её нет.

## 3. Алгоритм (5 шагов)

1. **Проба состояния.** `mcp__health-kb__kb_search` за ~3–7 дней: сон/recovery/strain (Whoop), настроение, свежие `05_decisions` (*что выбирала*). Свернуть в состояние + тренд.
2. **Якоря рамки.** Прочитать [[Обо мне]] (активные запросы, Фокус, ценности) + страницы с тегом `рамка` (ментальные модели / скрытые правила / цели).
3. **Детектор расхождений (триггер).** Найти конкретную улику конфликта поведения/исхода (health-kb) с рамкой/правилом/целью (wiki+профиль). Типы улик:
   - поведение ↔ ценность;
   - повтор ↔ скрытое правило;
   - происхождение цели («в чьей логике важно?»);
   - разрыв знание→действие (есть практика ровно под это, не используется).
   Нет улики → шаг 4-«рамка» пропускается.
4. **Сборка дайджеста** (см. §5 шаблоны).
5. **Приватность.** Применить §2.6.

## 5. Формат вывода

**Тихий (улик нет) — 2–3 строки:**
```
Состояние: <факт + тренд>.
Тактика (single-loop): <≤1 строка, опционально>.
Scout: <1 источник под Фокус, с реальной ссылкой> — почему тебе.
```

**Сработал (есть улика):**
```
Состояние: <факт + тренд>.
Рамка (double-loop): <прямое называние расхождения по улике>.
  Вопрос: <раздражающий вопрос к рамке/цели/правилу>.
  Рефрейм: [[страница]] — как она это переосмысляет.
  Выбор: (а) … / (б) … (один из вариантов может быть «бросить это»).
Scout: <источник, бьющий по этой рамке> — почему.
[Поднять забытое: <1 строка, только если кормит рамку>]
```

**Scout on-demand** (`найди источники по X`): шортлист 5, по каждому — формат (статья/эссе/книга/видео/лекция/сайт) · почему тебе · ось пользы · приоритет · ⚠️ спорное/хайп. Выбранное → `raw/inbox.md` (книги — в `_справочник.md` по правилу). Названия/ссылки не выдумывать (граунд реальным поиском).

## 8. Критерии успеха

- При наличии улики дайджест называет расхождение, которого Кристина сама не сформулировала (не «устала → отдохни»).
- Без улик — короткий и не лезет в рамку.
- Ни один health-факт не попал в `wiki/`/git.
- Scout приносит реальные (проверяемые) источники, привязанные к рамке/Фокусу, разного формата.

## 9. Вне скоупа

- Автоматический анализ корреляций как payload (это single-loop; паттерны — только как улика к рамке).
- Запись/изменение данных в health-kb (только чтение).
- Автопубликация Scout-находок в вики без ревью (выбор — за Кристиной).
