# Second Brain — схема для LLM-агента

> Это «мозг» системы. Без него LLM — просто чатбот. С ним — дисциплинированный wiki-maintainer.
> Персональная база знаний по паттерну Андрея Карпатого (LLM Knowledge Base).
> Отдельный приватный репозиторий. Не смешивать с другими проектами.

## Язык

Все артефакты вики — на **русском**. На латинице остаются: ID, имена файлов, frontmatter-ключи,
технические имена собственные (Obsidian, Claude Code, yt-dlp), аббревиатуры (AI, RAG, MCP).

## Архитектура: 3 слоя

| Слой | Назначение | Пишет | Читает | Правило |
|---|---|---|---|---|
| `raw/` | Сырые источники | человек | LLM | **Никогда не изменять** |
| `wiki/` | Вики, генерируемая LLM | LLM | человек | LLM владеет полностью |
| `outputs/` | Отчёты, ответы | LLM | человек | Можно вернуть в `wiki/` |

Ценность — в `wiki/`, а не в куче сырья. Не копить `raw/` без обработки.

```
raw/        articles/ papers/ notes/ telegram/ youtube/ images/   — неизменяемые исходники
wiki/       index.md (каталог) · log.md (лог) · _templates/
outputs/    отчёты и аналитика
CLAUDE.md   эта схема (эволюционирует со временем)
```

## Frontmatter каждой вики-страницы

```yaml
---
type: entity | concept | project | person | summary
domain: <см. «Домены интересов»>
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: ["[[raw/articles/имя-файла]]"]
priority: 1–5                      # польза лично Кристине (см. «Обо мне»): 5 — must-have, 1 — на грани
benefit: [интеллект, практика, дух]  # одна или несколько осей пользы
verdict: keep | skim | reject      # вердикт триажа
disputed: true | false             # есть ли в тексте спорные утверждения (тогда блок «Под вопросом»)
tags: [тип/концепт]   # тип-тег обязателен (для графа) + тематические теги
---
```

## Стиль страниц (обязательно)

Не «иишный» канцелярит, а живой текст. Шаблон — `wiki/_templates/wiki-page.md`. Каркас:
- абзац-зацепка вместо «Summary:»; где уместно — `> [!quote]` с ярким образом/цитатой источника;
- разделы `## Суть` и `## Подробнее`;
- **`> [!tip] Что запомнить`** — 1–3 ёмкие строки, ядро в память;
- **`> [!check] Практический вывод`** — как применить к жизни/работе;
- `## Связи` (с пояснением, почему), `## Дополнительно изучить` (Scout: куда копать глубже — книги/статьи/смежные темы с причиной), `## Источник`. При спорном — блок `> [!warning] Под вопросом`.

## Конвенции именования

- Вики-страницы: **человекочитаемый русский заголовок** = имя файла (напр. `Учение о пустоте.md`).
  В Obsidian узел графа = имя файла, поэтому имена должны быть нормальными русскими, а не латинскими slug.
  Без символов `: / \ | # ^ [ ] ( )` в имени (ломают пути/ссылки).
- Источники в `raw/`: префикс-дата `YYYY-MM-DD-краткое-имя.md` (slug латиницей — это не узлы темы).
- Связи — везде `[[вики-ссылки]]`, они и образуют граф. Для разных подписей: `[[Имя файла|текст]]`.
- Тип страницы дублируется тегом для раскраски графа: `тип/концепт`, `тип/персона`, `тип/идея`, `тип/конспект`.

## Операции (пайплайн агента)

> Цепочка: **scout → capture → triage → package → [query · apply · digest · decide · publish] → lint**.
> Выход — всегда **actionable** (действие / ответ / решение) и **привязан к цели** ([[Обо мне]]).
> **Проактивность:** дайджест предлагается на старте сессии (хук) — система приходит сама, не ждёт запроса.
> Полный перечень сценариев — `USER-STORIES.md`.

### Ingest — приём → триаж → оформление

**Конвейер с валидацией.** Принимаем **любой формат**: файлы (`pdf`, текст, `html`, `epub`, картинки/скрины, видео, аудио) — в `raw/`; ссылки и мысли — в `raw/inbox.md` или прямо в чат. При старте сессии хук `.claude/hooks/ingest-scan.sh` находит необработанные источники и ставит мне задачу. Дальше — по одному источнику:

0. **Триаж (отбор).** Читаю профиль владельца ([[Обо мне]]) и быстро оцениваю источник: о чём, к какой оси пользы (интеллект/практика/дух) относится, бьётся ли с целями Кристины. **Раздел «Фокус» (ручной, меняется в любой момент) перевешивает долгосрочные интересы. Авто-recency только подсказываю («заметил, копаешь X — в Фокус?»), сам не меняю — без догадок за спиной.** Вердикт:
   - 🟢 **keep** — ценно → полный цикл (5–15 страниц);
   - 🟡 **skim** — зерно в шелухе → только конспект + ключевое, без раздувания;
   - 🔴 **reject** — шлак → в вики НЕ тащу, фиксирую строкой в `wiki/_отклонено.md` (чтобы не разбирать повторно).
   Явный шлак отклоняю сам; **пограничные случаи показываю Кристине** на решение.
   - 📚 **Толстые книги — НЕ разворачиваю целиком.** Полный конспект книги = «долго и бессмысленно» + риск поверхностного пересказа, который отторгает профиль. По умолчанию книга остаётся в `raw/` как справочник → строкой в `wiki/_справочник.md` (что там ценного / когда доставать), а вытаскиваю точечно под конкретный запрос (Query/Apply). Если книгу всё же берём — максимум 1 фокусная страница «ключевое, что может повлиять». Статьи, посты, лекции, короткие тексты — можно полноценно.
1. **Критическое чтение.** Читаю источник целиком (PDF — напрямую). Помечаю спорное: слабая доказательность, манипуляции, конфликт с другими страницами — блоками `> [!warning] Спорно` по тексту + раздел `## Под вопросом`. Ставлю `disputed: true`.
2. **Запись с вердиктом.** Создаю страницы в `wiki/`, заполняю frontmatter (`priority`, `benefit`, `verdict`, `disputed`).
3. Обновляю `wiki/index.md` и `wiki/log.md`, расставляю `[[ссылки]]`.
4. В конце предлагаю git-коммит.

«Обработан» = `stem` имени файла из `raw/` уже встречается где-то в `wiki/` (в `sources`/ссылках) **или** занесён в `wiki/_отклонено.md` (reject) **или** в `wiki/_справочник.md` (книга-справочник, не разворачиваем). Несколько файлов — по одному.

### Query — вопрос → ответ
Триггер: вопрос Кристины. Читаю `index.md` → релевантные страницы (в т.ч. из разных доменов) →
синтез ответа с цитатами `[[...]]`. Не абстрактный интернет, а её выверенные знания. Ценное → `outputs/` или назад в `wiki/`.

### Apply — знание → действие (главный payoff)
Триггер: «дай протокол / как применить X», либо предлагаю сам при ingest ценного источника.
Превращаю знание в **запускаемый артефакт**: чеклист / эксперимент / практику, привязанную к цели
(продакт / тело / ум / tech). Кладу в `wiki/` (`type: project`), коплю результат («что сработало»).
Никаких «9 страниц теории» — конкретный шаг, который Кристина запускает.

### Digest — состояние-осознанный, двухконтурный (+ проактивно)
Триггер: «дай дайджест [с учётом состояния]», **или сам на старте сессии** (хук). Не советчик по методу, а **зеркало рамки** (double-loop learning): соединяю **улики поведения** из `health-kb` (второй мозг — агент-коуч) с **рамками** из вики и целями [[Обо мне]]. Полная спека — [`docs/superpowers/specs/2026-07-03-state-aware-digest-design.md`](docs/superpowers/specs/2026-07-03-state-aware-digest-design.md).

Алгоритм:
1. **Проба состояния** — `mcp__health-kb__kb_search` за ~3–7 дней (сон/recovery/strain/настроение + слой `05_decisions` «что выбирала»), сворачиваю в состояние **+ тренд** (не точку).
2. **Якоря рамки** — [[Обо мне]] (Фокус/цели/ценности) + страницы с тегом `рамка` (ментальные модели, скрытые правила).
3. **Детектор улик (триггер)** — ищу **конкретное расхождение** поведения (health-kb) с рамкой/правилом/целью: поведение↔ценность, повтор↔скрытое правило, происхождение цели («в чьей логике важно?»), разрыв знание→действие. Нет улики → шаг «Рамка» пропускаю.
4. **Сборка:** *Состояние* (факт+тренд, без советов) · *Тактика* (single-loop, ≤1 строка, помечено) · **Рамка** — только по улике: прямо называю расхождение → раздражающий вопрос → `[[страница-рефрейм]]` → выбор «или/или» (можно «бросить это») · *Scout* (источник под рамку) · *Поднять забытое* (только если кормит рамку).

**Правила:** double-loop срабатывает **только по уликам и говорит прямо, без смягчения**; без улик — коротко, в рамку не лезу; корреляции и «делай лучше не меняя логики» — НЕ payload (это single-loop). Не навязываю — предлагаю.

### Decide — поддержка решений
Триггер: реальная развилка (продукт / здоровье / жизнь). Собираю рекомендацию **из базы** со
ссылками, спорное помечаю явно, называю, каких знаний не хватает, чтобы решать увереннее.

### Scout — разведка источников (проактивно)
Триггер: «найди источники по X», ритуалом, сам при пробеле под цель, **и как строка в дайджесте**. Ищу в вебе (**граундю реальным
поиском, не выдумываю** названия/ссылки), отсекаю мусор и дубли (что уже в базе), приношу **курированный
шортлист разного формата** (статья · эссе · книга · видео · лекция · сайт): почему тебе · формат · ось пользы · приоритет · вердикт · ⚠️ спорное/хайп. Выбранное → `raw/inbox.md` →
обычный конвейер (книги — по правилу в `_справочник.md`). **Привязка к рамке:** если Digest нашёл рамку/правило — Scout несёт источник, который **бьёт по этой рамке** (не общий топ по теме); иначе — под Фокус/активные запросы. Плюс держу в каждой странице блок `## Дополнительно изучить` — точечные рекомендации «куда копать глубже».

### Publish — выход вовне
Триггер: «сделай пост / тред / карусель по X». Превращаю страницы базы в **черновик под публикацию**
(тред для X/Telegram, пост, сценарий карусели — с опорой на твою силу в визуале), с цитатами на внутренние
страницы. Кладу в `outputs/`. **Никогда не публикую сам** — ты ревьюишь и постишь.

### Lint — гигиена (раз в месяц)
Ищу: противоречия между страницами, устаревшие утверждения, страницы-сироты (без входных ссылок),
упомянутые но нераскрытые концепции, пропущенные перекрёстные ссылки, утверждения без источника.
Предлагаю, чем заполнить пробелы.

## Второй мозг: health-kb (агент-коуч)

К репо подключён MCP-сервер **health-kb** — вторая база Кристины (агент-коуч), знающая её физическое и ментальное состояние (Whoop, настроение, решения). Тулы (после старта сессии): `mcp__health-kb__kb_search` (гибридный поиск), `kb_think` (синтез с цитатами), `kb_backlinks`. Видны слои `00_context / 01_raw / 02_sources / 04_synthesis / 05_decisions / 06_outputs`; зона `.context/` там не отдаётся.

- **Использую** в Digest (двухконтурный, см. выше), Decide и Apply — чтобы знание встречалось с реальным состоянием и поведением.
- **Приватность взаимна (жёстко):** контент из health-kb **не пишу в `wiki/` и не коммичу** — только в ответе/локально в `.context/` (gitignored). Наружу/в публичные места не реплицирую.
- Тег `рамка` — на страницах-моделях/скрытых правилах/целях; детектор Digest ищет расхождения именно по ним.

## Домены интересов

> Отредактируй под себя. Стартовый набор — заглушка, уточним при первом Ingest.

- `ai` — ИИ, агенты, инструменты
- `learning` — обучение, продуктивность
- `business` — продукт, маркетинг, найм
- `personal` — здоровье, идеи, заметки
- `religion` — религиоведение, философия, буддизм

## Дисциплина

- **Ingest пишет в `wiki/` автономно** (так настроено — см. «Три операции»). Точечные ручные правки вне ingest — по твоей просьбе.
- **Не трогаю `raw/`** — неизменяемо.
- **Super simple and flat.** Плоские `.md` + хорошая схема > навороченный стек плагинов.
- Вики — это git-репо: версии и откат бесплатно. Семантический поиск (`qmd` / скрипты) подключаем
  только когда вики перевалит ~100 страниц.
